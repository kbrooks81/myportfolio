Case StudyMovieFlix

React Native • Expo • TMDB API • Appwrite

Summary

MovieFlix is a React Native movie discovery app built with Expo and file-based routing. It focuses on a clean, mobile-first browsing experience for exploring titles, opening detailed views, and keeping navigation predictable as users move between screens. It utilizes an Appwrite backend for saving movie favorites.

Live demo: A full walkthrough is available below showcasing the user flow from homescreen → movie details → movie search → movie favorites.

At a glance

  • Platform: React Native (Expo)
  • Routing: File-based routing via Expo Router
  • Core skills: Component-driven UI, typed data models, screen navigation patterns
  • Key features: Browse titles, search/discover flow, detail screens, reusable UI components, favorite movies
  • Deliverable: Mobile app source code + live demo

Problem

Movie apps can feel cluttered fast: too many screens, inconsistent navigation, and detail pages that do not clearly communicate what matters (rating, overview, metadata). I wanted to practice building a clean discovery flow with predictable routing and a UI structure that scales as screens/features grow.

Solution

MovieFlix uses Expo + file-based routing to keep navigation straightforward and maintainable. The app is organized around reusable UI components and typed interfaces so screens remain consistent and data is handled safely as it moves through the app.

Key features

  • Discovery flow: browse and explore titles with a mobile-first layout
  • Detail screens: focused presentation of a titles key information
  • Reusable components: shared UI building blocks for consistent screens
  • Favorites Database: users personal favorite movie list stored using an Appwrite backend.
  • Navigation structure: file-based routing for a clean, scalable screen map

Screenshots

Challenges and tradeoffs

  • Navigation design: keeping routes predictable as screens expand
  • UI consistency: ensuring cards, spacing, and typography remain uniform across screens
  • Data boundaries: maintaining clear contracts between services, components, and screens
  • Mobile UX: balancing information density with readability and touch targets
