Case StudyTest-Driven Development

Python • Django • Automated Testing

Summary

This project was completed as part of a university-level course focused entirely on Test-Driven Development (TDD). Using Python and Django, the application was built incrementally by writing automated tests first and implementing code only to satisfy failing tests.

No traditional demo: This project emphasizes development methodology, test design, and iterative refinement over a polished UI or standalone application.

At a glance

  • Type: Test-Driven Development coursework
  • Language: Python
  • Framework: Django
  • Testing tools: unittest, Django test framework, Selenium
  • Source location: GitHub Repository

Problem

The challenge was not simply to build a working application, but to rigorously apply Test-Driven Development principles throughout the entire lifecycle. Each feature needed to be justified by a failing test, with the final system emerging through small, verified steps.

Goals

  • Discipline: write tests before production code
  • Reliability: ensure changes are validated through automated tests
  • Maintainability: keep code modular and testable
  • Confidence: refactor safely using test coverage

Solution

Following Test-Driven Development with Python by Harry Percival, the project progressed through a strict Red → Green → Refactor cycle. Functional tests were written to describe user behavior, followed by unit tests that drove internal design decisions.

Key features

  • Functional tests: browser-level tests using Selenium
  • Unit tests: fine-grained testing of application logic
  • Django integration: test-driven views, models, and templates
  • Incremental design: features evolve only as tests require them
  • Refactoring: continuous cleanup guided by passing tests

TDD methodology

Red

Write a failing test that describes the desired behavior from the users perspective.

Green

Implement the minimum amount of code needed to make the test pass.

Refactor

Improve structure, readability, and design while keeping all tests passing.

Design & structure

  • Tests: functional tests and unit tests drive all features
  • Application: Django project built incrementally
  • Philosophy: tests describe behavior before implementation
  • Outcome: a resilient codebase with high confidence in changes

What I would improve next

  • Coverage: expand edge-case and negative testing
  • CI integration: automated test runs on every commit
  • Performance testing: validate scalability under load
  • Documentation: clearer mapping between tests and requirements
