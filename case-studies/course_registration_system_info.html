Case StudyCourse Registration System

Java • Command Line • File-Based Data

Summary

This Course Registration System (CRS) is a Java command-line application built as a school final project. It simulates a basic registration workflow using a menu-driven interface and file-based inputs for course and student data.

No live demo: This project runs in the terminal. This case study focuses on the problem, design decisions, and example CLI runs.

At a glance

  • Type: Java CLI application (school assignment)
  • Language: Java
  • Data inputs: CourseList2.txt, StudentList.txt
  • Source location: GitHub Repository

Problem

The goal was to build a command line system that models course registration with realistic constraints: users should be able to view available courses, register and drop courses, and see their current schedule— while the application prevents invalid actions and handles poor input gracefully.

Goals

  • Clarity: a clean, menu-driven CLI flow that is easy to follow
  • Correctness: consistent rules for registration and dropping
  • Safety: strong input validation to avoid crashes and bad state
  • Maintainability: keep business logic separated from the CLI prompts

Solution

CRS uses a simple loop-based CLI menu to guide users through common registration actions. Course and student data are loaded from text files, then the program applies validation and registration rules before updating the in-memory state and printing user-friendly results back to the terminal.

Key features

  • Role-based access: separate Admin vs Student login flows
  • Course management (Admin): add and delete courses
  • Student management (Admin): enroll students, unenroll students, drop a course from a student
  • Registration (Student): register for a course using course ID + student name verification
  • Reporting (Admin): view course details and student list / enrollment reporting
  • Validation rules: max 5 courses per student, no duplicate registrations, cannot register if course is full

Roles & business rules

Roles

  • Admin: manages courses and student enrollments, and runs reports
  • Student: registers for courses through the CLI menu

Business rules & validations

  • Max course load: a student can register for up to 5 courses
  • No duplicates: a student cannot register for the same course twice
  • Capacity constraint: if a course is full, registration is blocked

Data & design

CRS is built as a menu-driven Java CLI application with file-based inputs for course and student records. The CLI layer handles prompts and input parsing, while the registration logic enforces role permissions and business rules (capacity, duplicates, and max course load).

  • Inputs: CourseList2.txt, StudentList.txt
  • Source: Java code under src/crs
  • CLI layer: menu loop, prompts, input validation, user messaging
  • Logic layer: course/student operations, registration rules, reporting output
