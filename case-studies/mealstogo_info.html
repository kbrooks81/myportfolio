Case StudyMealsToGo

React Native • Expo • Context API

Summary

MealsToGo is a React Native restaurant discovery app that lets users search a city, browse restaurants, and explore locations on a map with a smooth, predictable selection flow between tabs.

Live demo: A full walkthrough is available on YouTube showing the user flow from city search → list → map → selection.

At a glance

  • Platform: React Native (Expo)
  • Core skills: Context API state management, navigation, map interactions, UX consistency
  • Key features: City search, viewport-based map zoom, selection syncing
  • Deliverable: Mobile app experience + recorded live demo

Problem

Restaurant discovery experiences often break down when users change cities: map zoom can feel wrong, previously selected restaurants can linger, and the UI can become inconsistent across list and map views. I wanted a flow where user intent is always reflected immediately and correctly.

Solution

MealsToGo uses a city-first state model and predictable “selection reset” behavior. Searching a new city updates the viewport and clears stale restaurant selection so the map and list stay aligned.

Key features

  • City search: Google Places autocomplete for fast, accurate location selection
  • Map behavior: viewport-based zoom to match the selected city
  • Selection syncing: selecting a restaurant updates map markers + detail UI
  • State hygiene: selection resets on city changes to prevent stale UI

Screenshots

Challenges and tradeoffs

  • State consistency: ensuring city changes clear stale restaurant selection
  • Map UX: keeping zoom/viewport feeling natural across different cities
  • Performance: avoiding unnecessary re-renders when switching tabs or locations
  • Mobile layout: balancing readable cards with touch targets and scroll behavior
